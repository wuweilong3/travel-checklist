# 部署指南

## 部署到公网

### 方案：Vercel（前端）+ Railway（后端）

---

## 第一部分：部署后端到 Railway

### 1.1 创建 Railway 账号

1. 访问 https://railway.app
2. 使用 GitHub 账号登录
3. 进入 Dashboard

### 1.2 部署后端

1. 点击 "New Project" → "Deploy from GitHub repo"
2. 选择你的仓库（如果没有，先上传代码到 GitHub）
3. Railway 会自动检测为 Python 项目
4. 配置环境变量：
   - `DASHSCOPE_API_KEY` = 你的阿里云百炼API密钥
   - `LLM_PROVIDER` = dashscope
   - `MODEL_NAME` = qwen-plus

5.  Railway 会自动执行 requirements.txt 安装依赖

### 1.3 获取后端URL

部署成功后，在 Railway 项目设置中找到你的后端URL，格式类似：
```
https://your-backend.railway.app
```

---

## 第二部分：部署前端到 Vercel

### 2.1 创建 Vercel 账号

1. 访问 https://vercel.com
2. 使用 GitHub 账号登录

### 2.2 部署前端

1. 点击 "Add New" → "Project"
2. 导入 `frontend` 文件夹（或整个项目仓库）
3. 配置项目：
   - Framework Preset: Vite
   - Root Directory: `./` (如果是monorepo选择frontend)
   - Build Command: `npm run build`
   - Output Directory: `dist`

4. 配置环境变量：
   - `VITE_API_URL` = 你的Railway后端URL

### 2.3 修改前端API配置

在 `src/services/api.ts` 中修改 baseURL：

```typescript
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL + '/api',
  // ...
});
```

### 2.4 更新 Vercel 配置

修改 `vercel.json`：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "dist"
      }
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "https://your-backend.railway.app/api/$1"
    },
    {
      "src": "/(.*)",
      "dest": "/index.html"
    }
  ]
}
```

### 2.5 部署

1. 点击 "Deploy"
2. 等待构建完成
3. 获取 Vercel 分配的域名，例如：`your-app.vercel.app`

---

## 第三部分：配置自定义域名（可选）

### 后端（Railway）
1. 进入 Railway 项目设置
2. 选择 "Settings" → "Networking"
3. 点击 "Generate Domain"
4. 或添加自定义域名

### 前端（Vercel）
1. 进入 Vercel 项目设置
2. 选择 "Domains"
3. 添加你的自定义域名
4. 配置 DNS 记录

---

## 快速部署脚本

### 后端 Railway (railway.toml)

```toml
[railway]
  pythonVersion = "3.10"

[build]
  builder = "python"

[deploy]
  startCommand = "uvicorn main:app --host 0.0.0.0 --port $PORT"
  healthcheckPath = "/health"
```

---

## 注意事项

1. **CORS配置**：后端需要配置 CORS 允许 Vercel 域名访问
   - 在 `backend/main.py` 中添加：
   ```python
   app.add_middleware(
       CORSMiddleware,
       allow_origins=["https://your-app.vercel.app"],
       allow_credentials=True,
       allow_methods=["*"],
       allow_headers=["*"],
   )
   ```

2. **环境变量**：确保在 Railway 中正确配置所有环境变量

3. **API Key 安全**：不要将 API Key 提交到 GitHub，使用 Railway 的环境变量功能
